char - 문자형
int - 정수형
float - 실수형

자료형은 이름에서 나타내듯이 대응되는 타입이 존재한다. 위에 적혀있는 대로 말이다. 하지만 그건 나타내기에 적합하다는 뜻이지 절대적인 건 아니다. 메모리에 대한 이해 부족이다.

메모리는 공간일 뿐 어디 귀퉁이에 '여긴 문자형.. 저긴 정수형...' 이렇게 적어두지 않는다.

그렇다면 자료형은 어떻게 정의하면 되는가? 

크기다. 그저 크기일 뿐이다. int의 4바이트를 쪼개서 char 타입의 크기가 4인 배열로 사용할 수도 있다. 믿기지 않는가?

{% highlight cpp %}
#include <cstdio>

int main()
{
	int arr;
	char* pArr = (char*)& arr;
	pArr[0] = 'a';
	pArr[1] = 'b';
	pArr[2] = 'c';
	pArr[3] = 'd';

	for (int i = 0; i < 4; i++)
	{
		printf("%c \n", pArr[i]);
	}

	return 0;
}
{% endhighlight %}

실행 결과는 굳이 말하지 않아도 뻔하다. 자료형에 대응되는 크기는 시스템마다 천차만별이다. 

char는 1Byte, int는 4Byte 같은 식으로 외우면 절대 안 된다. 공학은 종교가 아니다. 새로운 언어나 시스템에서 프로그래밍을 할 일이 생긴다면 반드시 자료형의 크기를 확인해야 한다. 특히 문자열은 여러 가지 인코딩이 존재하기 때문에 주의해야 한다.

관리 언어를 사용하다 보면 메모리에 대한 개념이 희미해진다. 하지만 본능적으로 알고 있어야 한다. 메모리는 그저 공간의 나열일 뿐이고 그 안의 데이터를 어떻게 표현하냐에 따라 달라진다는 것을

아 참, 실수형은 표현 방식이 완전히 다르다. 이것도 컴퓨터를 이해하는데 아주 도움이 된다.
