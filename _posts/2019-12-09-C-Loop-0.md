{% highlight cpp %}
#include <stdio.h>

int main()
{
	int sum = 0;
	int i;

	for(i = 1; i <= 10; i++)
	{
		sum += i;
	}

	printf("%d\n", sum);
}
{% endhighlight %}

1부터 10까지 더한 값을 출력하는 프로그램이다. 이 반복문이 정확하게 어떤 순서로 반복하는지 살펴보자.

![0](/assets/images/C-Loop-0/0.png)
​
가장 먼저 하는 일은 반복문에 사용될 변수를 초기화하는 것이다. mov [ebp+i], 1 명령어를 가장 먼저 실행하는 것을 볼 수 있다. 그리고 그 밑에 401016주소의 코드를 실행하지 않고 40101F의 코드를 실행한다. 여기서 401016영역은 반복 문의 카운터 변수를 증가시키는 영역이다. i++에 해당한다. 40101F는 비교와 내부 코드에 해당한다. 간단하게 반복 문의 처리 순서를 정리해보겠다.

1. 반복문에 사용되는 카운터 변수를 초기화한다.

2. 반복 여부를 결정짓는 조건을 검사한다.

3. 조건이 참이라면 내부 코드를 실행한다.

3. 조건이 거짓이라면 반복문을 빠져나간다.

4. 카운터 변수를 조작한다.(증가 또는 증감)

5. 2번으로 돌아간다.

생각보다 이러한 반복 문의 정확한 메커니즘에 대해 알지 못하는 경우가 많다. 이렇게 정확하게 알고 있게 되면 좀 더 다양하게 응용해서 사용할 수 있다. for(;;)가 무한 반복문 역할을 하는 것처럼 반복문은 아주 다양하게 활용할 수 있다. 일단 반복문에 대해서 좀 더 살펴보기로 하자.

**continue를 사용하면 카운터 변수가 증가할까?**

바로 대답하지 못하고 잠깐 생각한 사람도 있을 것이고 바로 대답한 사람, 또는 모르는 사람도 있을 것 같다. 그렇다면 반복 문의 메커니즘을 이해하고 논리적으로 대답한 사람은 얼마나 될까? 위의 내용을 보고 생각해보자. continue는 반복 문의 내부에 존재하는 코드다. 위의 순서에 따라 코드를 실행하고 변수를 조작한다. continue는 뒷부분의 실행을 생략하고 바로 카운터 변수를 조작하는 부분으로 점프하기 때문에 카운터 변수의 값이 변하게 된다.

**반복 문의 식에서 ++i가 i++보다 빠른가?**

상당히 많은 사람들이 반복문에서 ++i가 더 빠르다고 생각한다. 아직도 구글링을 하면 쉽게 찾아볼 수 있다. 그리고 대부분 이 말을 덧붙이는데 '컴파일러의 최적화 때문에 이제 ++i와 i++은 거의 차이가 없다.' 이는 잘못된 내용이다. 최적화로 속도 차이가 거의 안 나는 게 아니라 애초에 같다. 한번 테스트를 해보자. 컴파일러의 최적화를 사용 안 함으로 변경한 후 테스트하였다.

{% highlight cpp %}
#include <stdio.h>

int main()
{
	int sum = 0;
	int i;

	for(i = 1; i <= 10; i++)
	{
		sum += i;
	}

	for(i = 1; i <= 10; ++i)
	{
		sum += i;
	}
}
{% endhighlight %}

이 코드를 컴파일한 후 어셈블리를 확인해보겠다.

​![0](/assets/images/C-Loop-0/1.png)

똑같은 코드가 두 번 반복되는 것을 볼 수 있다. 아무런 차이가 없다. 이로써 반복문에서의 전위, 후위 연산자는 아무런 차이가 없다는 것을 확인했다. 사실 ++i가 더 빠르다고 생각하고 프로그래밍 해도 달라질 건 없다. 그래도 아닌 건 아닌 거다.
