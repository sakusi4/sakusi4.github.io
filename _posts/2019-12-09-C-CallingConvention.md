C언어를 공부하면서 호출 규약 부분이 나오면 제대로 보지 않는 경우가 많다. 당장 모르더라도 프로그래밍을 하는데 영향을 주지 않기 때문이다. 그럼에도 아는 것이 프로그램에 대한 이해도를 높일 수 있다. 호출 규약과 함께 이론으로만 들었던 스택 영역이 어떤 방식으로 작동하는지 알아보도록 하겠다.

호출 규약에 대해 알아보기 전에 스택에 대해 먼저 살펴보도록 하자. 대표적인 호출 규약인 cdecl과 stdcall의 차이는 스택을 정리하는데 차이점이 있기 때문이다. 이를 위해 간단한 코드를 작성해보자.

{% highlight cpp %}
#include <stdio.h>

void swap(int * pA, int * pB)
{
	int temp;
	temp = *pA;
	*pA = *pB;
	*pB = temp;
}
int main()
{
	int a = 10;
	int b = 20;

	printf("before swap, a = %d, b = %d\n",a,b);
	swap(&a,&b);
	printf("after swap, a = %d, b = %d\n",a,b);

	return 0;
}
{% endhighlight %}

두 개의 변수의 값을 교환하는 프로그램이다. 어떤 방식으로 메모리를 사용하는지 확인해보자.

​![0](/assets/images/C-CallingConvention/0.png)

프로그램에서 사용하는 함수는 두 개다. main과 swap 함수, 위 코드는 main 함수의 코드다. main 함수의 스택은 어떤 모양인지 확인해보자. 

{% highlight nasm %}
push ebp
mov ebp, esp
////생략////
mov esp, ebp
pop ebp
retn
{% endhighlight %}

위 코드와 같은 형태를 함수의 프롤로그와 에필로그라고 한다. 프로그램에선 함수마다 독립적인 스택 영역이 필요한데 실제 메모리 공간은 그러하지 못하다. 함수마다 개별적으로 스택이 존재하는 게 아니라는 뜻이다. 하지만 우리는 지역 변수는 다른 영역에서 참조할 수 없다고 배웠고 실제로 아무런 의심 없이 다른 영역의 지역변수에는 접근하지 않는다. 그렇다면 같은 스택 영역을 사용하는데 어째서 접근이 불가능한 것일까? 대답할 수 있다면 뒤로 가기를 누르길 바란다. 

비밀은 위 코드의 형태에 있다. esp는 스택의 꼭대기를 가리키는 레지스터인데 이는 PUSH와 POP이 이루어질 때마다 변경된다. 그렇다면 어디까지가 이 함수의 변수이고 다른 함수의 변수인지 구분이 불가능하다. 말했다시피 스택은 하나인데 모든 지역 변수가 이 스택에 저장되기 때문이다. 이 때문에 ebp라는 하나의 레지스터를 더 사용한다. 함수를 시작할 때에 현재 ebp 값을 저장(백업) 해두고 ebp에 esp 값을 넣어둔 뒤(백업) 자유롭게 PUSH와 POP을 한다. 그리고 함수를 끝낼 때에 ebp에 저장해둔 esp 값을 다시 가져오면 PUSH와 POP을 하면서 변경된 값을 복원할 수 있다.

​![0](/assets/images/C-CallingConvention/1.png)

main 함수의 첫 번째 줄인 push ebp가 실행되기 전의 스택이다. 현재 스택의 값은 004011EF로 오른쪽에 RETURN to swap.004011EF라는 코멘트가 달려있는데 저번 포스팅에서 살펴봤지만 main 함수는 프로그램의 진짜 시작점이 아니다. main 함수도 호출되는 함수기 때문에 끝이 나면 돌아가야 할 곳이 있다. 현재 스택에는 main 함수가 종료되면 돌아갈 곳이 저장돼 있다. 두 줄 더 실행시켜보도록 하겠다.

​![0](/assets/images/C-CallingConvention/2.png)

스택에 ebp 값을 PUSH 하면서 스택이 자라난 것을 확인할 수 있다. 스택은 높은 주소에서 낮은 주소로 자라난다. 0012FFC0는 이전 함수에서 사용하던 ebp 값 일 것이다. 함수를 종료하고 호출부로 돌아가게 되면 돌아간 함수에서도 정상적인 참조가 이루어져야 하기 때문에 당연한 수순으로 생각할 수 있다.

​![0](/assets/images/C-CallingConvention/3.png)

위의 어셈블리는 [ebp+a]와 같은 형식으로 표시하고 있어서 오해의 소지가 있는데 a의 값은 -4다. 위 이미지를 보면 알겠지만 지역변수를 사용하기 위해서 ESP에서 8(두 개의 변수기 때문에 8)을 뺀 뒤 8을 빼기 전의 값이 들어 있는 ebp에서 거리만큼 주소를 빼서 값을 할당한다. 이는 스택의 공간을 미리 확보한 뒤에 값을 넣어준 것이다. 바로 PUSH 0A, PUSH 14를 하는 것과 같다. 위 코드까지 실행하고 난 다음의 스택이다.

​![0](/assets/images/C-CallingConvention/4.png)

위에서 두 개의 함수를 사용한다는 말을 했는데 실제로는 printf 함수까지 하나 더 사용한다. 이 함수에도 마찬가지로 독립적인 스택 공간이 필요하다. printf 함수를 사용하는데 3개의 인자를 전달했으므로 세 번의 PUSH가 이루어진다. 그리고 printf 함수를 호출하는데 호출 직후의 스택의 상황을 확인해보자.

​![0](/assets/images/C-CallingConvention/5.png)

3번의 PUSH를 하고 함수를 호출했지만 실제로는 4번의 PUSH가 일어났다. 바로 복귀 주소를 호출 직전에 PUSH 한 것이다. 이 동작이 없다면 함수를 수행하고 돌아올 곳이 없기 때문에 문제가 발생하게 될 것이다. 이렇게 프로그램을 진행할수록 스택에 데이터가 차곡차곡 쌓여나가는 것을 볼 수 있다. printf 함수 호출을 완료한 후의 스택을 확인해보자.

​![0](/assets/images/C-CallingConvention/6.png)

함수 호출 후 add esp, 0C까지 수행한 모습이다. C는 12로 함수 호출 전에 3번의 PUSH를 했기 때문에 이로 인해 감소한 12만큼 다시 더해주는 것이다. 호출 직전에 PUSH 한 복귀 주소는 printf 함수의 마지막에 POP 된다. 이번에는 swap 함수의 호출 과정을 살펴보자.

​![0](/assets/images/C-CallingConvention/7.png)

printf 함수를 호출할 때처럼 swap 함수가 사용하는 두 개의 인자를 PUSH 한 뒤 함수를 호출하는 모습이다. 함수 호출 직후에 스택을 확인해보겠다.

​![0](/assets/images/C-CallingConvention/8.png)

두 개의 주솟값과 복귀 주소가 PUSH된 것을 확인할 수 있다. swap 함수의 어셈블리를 확인해보자.

​![0](/assets/images/C-CallingConvention/9.png)

swap 함수만의 스택 영역을 확보하기 위해 main 함수와 똑같은 형식의 프롤로그와 에필로그가 보인다. 프롤로그 부분을 스택 프레임을 확보한다고 말한다. 스택 프레임이란 함수마다 개별적인 스택 영역을 의미한다. 스택 프레임을 확보한 후 의 스택 프레임은 위의 상태에서 ebp가 추가된 형태일 것이다. main 함수에서 사용하던 ebp 값을 유지해놔야 복귀 시에 사용할 수 있기 때문이다. 함수의 에필로그 부분을 확인해보자.

​![0](/assets/images/C-CallingConvention/10.png)

pop ebp까지 실행한 상태의 스택이다. 이 함수에서 temp라는 지역변수를 사용하기 때문에 스택이 한 번더 늘어나긴 했지만 ebp에 원래의 스택의 주소가 저장되어 있었기 때문에 돌아오는데 아무 문제가 없다. mov esp, ebp로 esp를 원상복구 시킨 다음 pop ebp로 main 함수에서 사용하던 ebp를 복구시키고 retn을 함으로 main 함수의 swap 함수 호출 다음 코드로 이동하게 된다. 

여기 까지가 프로그램에서 스택을 사용하는 방식에 대한 설명이었다. 설명하는데 재주가 없고 그림을 그리기엔 귀찮기 때문에 정말 노가다 형식으로 설명했지만 이해가 잘 되었으면 좋겠다. 이제 호출 규약에 대한 설명인데 여기까지 이해했다면 아무것도 아니다.

함수를 호출하기 전에 인자를 PUSH 하고 호출이 끝나면 'add esp, 상수'와 같은 코드를 이용해서 스택을 원상 복귀 시켰다. 아 참고로 스택을 원상복귀 시킨다는 것에 안 쓰는 값을 초기화 시키는 과정은 없다. 어차피 나중에 가면 덮어 써질 것이기 때문에 성능을 낭비하는 것이라 생각하기 때문이다. 위의 예시들과 같이 호출 부에서 스택을 정리하는 방식을 cdecl 방식이라고 한다. stdcall은 반대다. 호출된 함수에서 retn을 하면서 동시에 스택을 정리하는 방식을 stdcall 방식이라고 부른다. 

swap 함수의 호출 규약을 바꿔서 간단하게 어셈블리 코드를 확인해보겠다.

{% highlight cpp %}
#include <stdio.h>

void __stdcall swap(int * pA, int * pB)
{
	int temp;
	temp = *pA;
	*pA = *pB;
	*pB = temp;
}
int main()
{
	int a = 10;
	int b = 20;

	printf("before swap, a = %d, b = %d\n",a,b);
	swap(&a,&b);
	printf("after swap, a = %d, b = %d\n",a,b);

	return 0;
}
{% endhighlight %}

​![0](/assets/images/C-CallingConvention/11.png)

아까와 다르게 함수를 호출한 뒤 스택의 주솟값을 더해주는 과정이 사라졌다. 그렇다면 swap 함수 내부의 변화를 보자.

​![0](/assets/images/C-CallingConvention/12.png)

바뀐 것은 마지막 RETN 8이다. RETN 뒤에 상수를 넣어서 스택을 정리하는 것이다. 주로 C언어에서 함수를 만들어서 사용한다면 별다른 호출 규약을 명시하지 않는 이상 cdecl 방식을 사용하게 된다. stdcall 방식은 대부분의 API에서 사용되는 방식이다. 왜냐하면 보통 API들의 인자의 개수는 정해져있기 때문에 호출된 쪽에서 스택을 정리하는 게 가능하지만 printf 함수만 해도 인자의 개수가 가변적이라 호출된 함수에서 인자의 개수를 확인해서 스택을 정리하는 것이 쉽지 않다. 반대로 호출부 에서는 전달하는 인자의 개수를 알고 있기 때문에 스택을 정리하는 것이 가능하다.
