포인터의 문법 자체는 따로 검색해 보면 쉽게 파악할 수 있습니다.
이 글은 더 자세한 이해를 위한 글이므로 기본 문법 설명은 생략합니다.

'포인터는 주소값을 가질 수 있고 접근할 수 있다.'

그렇다면 포인터의 크기는 어때야할까요? 주소값을 가질 수 있다고 했으니 최소, 최대 주소값을 포함할 수 있는 크기면 됩니다. 32비트 기준으로는 주소 범위가 4GB이기 때문에 4Byte(2^32)면 모든 주소를 가리킬 수 있습니다.

그러므로 포인터의 크기는 타입과 상관없이 4Byte입니다.

 
**4Byte라면 Int 변수로도 포인터를 사용할 수 있나요?**
>답은 주소값은 가질 수 있지만, 접근이 안됩니다. 일반적인 변수는 주소값에 간접 참조하는 기능을 제공하지 않아요.


**포인터의 크기가 모두 같다면 포인터에는 왜 타입이 있나요?**
>포인터의 타입은 가리키기 위한 게 아니라 접근하기 위한 것입니다. 주소값의 위치로 가서 몇바이트를 읽을지 알려주는 수단입니다. 재밌는 예제를 보여드릴게요.

```Cpp
void fun(char *arr)
{
    arr[0] = 'A';
    arr[1] = 'B';
    arr[2] = 'C';
    arr[3] = 'D';
}

int main()
{
    int arr = 0;
    fun((char*)&arr);

    for(int i=0; i<4; i++){
        cout << ((char*)&arr)[i] << endl;
    }

    return 0;
}
/*
OUTPUT
A
B
C
D
*/
```

위의 원리를 이해했다면 이 예제 코드도 이해가 가능합니다.
4Byte인 Int 변수를 쪼개서 크기가 4인 Char 배열로 사용한 것이죠.

이러한 포인터의 특성 때문에 타입을 모른다면 얼마나 읽어야할지 알 수 없으므로 참조가 불가능합니다. 대표적인 게 void 타입 포인터입니다. 위치만 가리킬 뿐 크기를 모르는 거죠.

이쯤에서 하나 말하고 싶은 게 있네요.

call by value, call by reference라는 말을 들어보셨나요? 포인터를 설명할 때 항상 나오는 용어들입니다. 저는 이 용어가 좀 불편합니다. 사실 알고 보면 call by value 뿐입니다. 전달하는 값이 주소값이든 그냥 값이든 구분할 이유가 없습니다. 모두 값이며, 복사됩니다.

위 예제에서 fun 함수의 매개변수 타입을 int로 바꿔볼게요.

```Cpp
void fun(int arr)
{
    char* parr = (char*)arr;

    parr[0] = 'A';
    parr[1] = 'B';
    parr[2] = 'C';
    parr[3] = 'D';
}

int main()
{
    int arr = 0;
    fun((int)&arr);

    for(int i=0; i<4; i++){
        cout << ((char*)&arr)[i] << endl;
    }

    return 0;
}
```
실행 결과는 같습니다. 이 경우는 call-by-reference인가요? 개념적으로는 그렇다고 할 수 있겠네요. 그렇다면 call-by-value가 아니라고 할 수 있을까요? 그렇지는 않습니다. 주소값을 int 타입으로 복사한 게 맞으니까요.

생각보다 포인터의 원리가 간단하죠?
이후 포스팅에서는 reference와 포인터의 차이 점에 대해서 올려보겠습니다.

감사합니다.
